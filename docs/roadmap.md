# Roadmap

이 문서는 실제 진행 순서를 기록합니다. 목표는 빠르게 화면만 만드는 것이 아니라, PDF 자료를 안정적으로 누적하고 오답 복습까지 이어지는 구조를 만드는 것입니다.

## Phase 0. 프로젝트 골격

상태: 진행 중

- [x] 프로젝트 디렉토리 생성
- [x] 문서 디렉토리 생성
- [x] README 작성
- [x] 확장형 디렉토리 구조 설계
- [x] 데이터 모델 초안 작성
- [x] PDF 처리 계획 작성
- [x] 분류 체계 작성
- [x] 친구용 무설치 Codex 학습 모드 문서화
- [x] Codex 학습 진행 지침 작성

## Phase 1. 초기 PDF 등록

목표: 4개 PDF를 원본 자료로 등록하고 페이지 단위 작업 목록을 만든다.

- [x] `sources/raw/`에 초기 PDF 4개 복사
- [x] 파일 크기와 페이지 수 기록
- [x] `data/catalog/sources.json` 생성
- [x] `data/catalog/chapters.json` 생성
- [x] `data/catalog/taxonomy.json` 생성
- [x] 각 PDF의 페이지별 처리 상태 생성

완료 기준:

- 모든 PDF가 sourceId를 가진다.
- 각 페이지가 독립 작업 단위로 추적된다.

## Phase 2. 페이지별 추출

목표: PDF를 한 페이지씩 읽을 수 있는 Markdown으로 만든다.

- [x] PDF 텍스트 추출 방식 결정
- [x] `4시간 근현대사 수업자료.pdf` 이미지형 페이지 여부 확인
- [x] `5시간 전근대사 수업자료.pdf` 이미지형 페이지 여부 확인
- [x] `sources/extracted/final-1/page-001.md` 생성
- [x] `sources/extracted/modern-4h/page-001.md` - `page-019.md` 생성
- [x] `sources/extracted/premodern-5h/page-001.md` - `page-036.md` 생성
- [x] 추출 실패/이미지 기반 스킵/항목별 검토 대기 목록 기록
- [x] 페이지별 메타데이터 추가

완료 기준:

- 각 페이지의 원문 또는 OCR 결과를 Markdown에서 확인할 수 있다.
- 특정 문제의 출처 페이지를 다시 열어볼 수 있다.

- [ ] 전근대사 자료 정제
- [ ] 근현대사 자료 정제
- [ ] 파이널1 정제
- [x] 파이널2 구조 확인 및 키워드 카드 정제
- [ ] 새 챕터 후보 기록
- [ ] 헷갈리는 비교 포인트 기록

현재 진행:

- [x] `파이널1.pdf` 1쪽 1-14번 정제
- [x] `파이널1.pdf` 2-8쪽 15-120번 정제
- [x] `파이널2.pdf` 1-9쪽 1-85번 키워드 카드 정제
- [x] `4시간 근현대사 수업자료.pdf` 구조 확인
- [x] `5시간 전근대사 수업자료.pdf` 구조 확인
- [x] 이미지 기반 선택/판단 문제 스킵 규칙 반영
- [x] 손글씨, 회전된 인쇄 텍스트, 화면 안 텍스트를 먼저 전사하는 규칙 반영
- [x] `4시간 근현대사 수업자료.pdf` 1-15쪽 및 17-18쪽 concept_note 초안 생성
- [x] `5시간 전근대사 수업자료.pdf` 텍스트 강의 및 16쪽 손글씨 concept_note 초안 생성
- [x] `5시간 전근대사 수업자료.pdf` 백지 테스트 self_test_card 초안 생성
- [x] 혼합 기출 이미지 페이지 항목별 추출
- [x] `4시간 근현대사 수업자료.pdf` 19쪽 multiple_choice 초안 5개 생성
- [x] `5시간 전근대사 수업자료.pdf` 혼합 기출 페이지 multiple_choice/chronology 초안 36개 생성

- 문항화 가능한 개념과 사료가 표시된다.

목표: 앱에서 쓸 수 있는 문항만 approved로 승격한다.

- [x] 데이터 스키마 검증기 작성
- [x] 중복 ID 검사
- [x] 출처 누락 검사
- [x] 정답 선택지 참조 검사
- [x] 승인 문항의 근거 Fact 연결 검사
- [x] 자연어 하네스 시나리오 테스트 문서화
- [ ] 챕터/태그 누락 검사
- [x] 정답 검증 완료 객관식 문항의 draft에서 approved 승격 적용
- [ ] 재사용 가능한 승격 스크립트 작성

완료 기준:

- `approved` 데이터만으로 퀴즈를 실행할 수 있다.
- 잘못된 문항은 자동으로 걸러진다.

## Phase 6. 퀴즈 앱 MVP

목표: 실제로 문제를 풀 수 있는 첫 경험을 만든다. 초기에는 별도 앱 실행보다 Codex 대화형 출제를 우선한다.

- [ ] `approved` 문항으로 Codex 대화형 퀴즈 진행
- [ ] 한 문제씩 출제
- [ ] 답변 전 정답 숨김
- [ ] 답변 후 채점과 해설
- [ ] 세션 요약
- [ ] 챕터 선택
- [ ] 자료 유형 선택
- [ ] 출처 페이지 표시
- [ ] 풀이 기록 저장
- [x] 오답 기록 JSONL 형식 초안 작성

완료 기준:

- 승인 문항을 Codex 대화로 풀 수 있다.
- 틀린 문제는 오답 기록으로 남는다.

- [ ] 반복 오답 표시
- [ ] 복습 예정 문제 큐
- [ ] 챕터별 정답률
- [ ] 사료 문제만 풀기
- [ ] 파이널 문제만 풀기
- [ ] 취약 태그별 드릴

완료 기준:

- 사용자는 오늘 다시 풀어야 할 문제를 볼 수 있다.
- 많이 틀린 챕터와 태그가 드러난다.
